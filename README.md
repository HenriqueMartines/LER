## Modulo 2 - Levantamento de Requisito(continuação)

- O que é Levantamento de Requisitos
    - Transformar as necessidades do negócio em informação técnicas

- Requisitos Funcionais e Não Funcionais
    - RF - O que o sitema vai fazer
    - RNF - Como o sistema vai fazer 

- Requisitos, Regras, restricões
    - Requisitos: A parte Técnica do sistema
    - Regras: as necessidades do Cliente
    - Restricões: limitação imposto pela, linguagem, leis, normas,

- Modelo de Documentação de Requisitos

|valor|descriçao|Exemplo|
| --- | ------ | ------ |
| ID  | código | RF-002 |
| Título | valor | Tela de login |
| Descrição| O ator deve ser Capaz de | O cliente conseguira fazer login|
| Prioridade | Valor da Prioridade | Alta |
| Regras Vinculadas | Outras Regras Relacionadas | RN-001- Banco de dados|

### Técnicas de Levantamento de Requisitos
- Briefing
- Brainstorm
- Questionario
- Entrevista
- Etnografia
- Workshop
- Prototipagem

### Briefing
- Coleta de Dados Inicial
- Não Existe um Formato Pré-Definido
- Permite Coleta de Diversas Informações do Cliente
    - Manual de identidade Visual (Cores, Logo, Slogan, Fontes ...)
    - Concorrentes
    - Prazos 
    - Orçamento
    - Identidade da Empresa (Missão , Visão e Valores)
    - Publico - Alvo

### Brainstorm
- Reunião Criativa com Objetivo de Solucionar um ou mais **Problemas**
- Podem ser Estruturadas ou Não Estruturadas
- Reunião Curta Com Foco Bem Definido 
- Ferramentas
    - Miro
    - Mentimenter
    - OneNote

### Questionario
- Perguntas Direcionadas ao Cliente ,  com objetivo de montar o escopo do projeto (Documnetação final do projeto)
- Formas de fazer o questionario 
    - Reunião com perguntas diretas (Qualitativas)
    - Formulario de Coleta de dados (Qualitativas)

### Entrevistas
- Instrumento de Coleta de dados com usuárioa do projeto
- Entrevista podem ser formais e informais
- Nessa Estapa e necessario um MVP ( produto minimo viavel)

### Etnografia 
- Observação da cultura da empresa ,  a form COMO EL atende e se relciona com seu publico  interno e externo.

### Workshop
- Tecnica de levantamento de requisitos que acontecem na apresentação de uma MVP , para um grupo seleto de pessoas (stakeholders) com intuito de absorver a reação do publivo ao produto/projeto.
- Eventos, Feiras e WorkShops de produtos especificos.

### Prototipagem
- Criação de um elemento de design para simulação de um projeto , com intuito de testar funcionalidades e elementos gráficos antes de começar a escrever código
- os produtos podem ser de 
    - baixa fidelidade (rafe): rascunho de um desenho pode ser feito a mão 
    - media fidelidade (rough): desenhos no computador para marcação de espaços e layouts, não precisa de texto nem cores
    - alta fidelidade: protótipo focado nas aplicações de funcionalidade iguais aos do projeto que serão desenvolvidos posteriormente em codigo.
